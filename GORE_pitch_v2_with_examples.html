GORE
Graph Of
Recursive Execution

A minimal Turing-complete language designed as a controlled training environment for backtracking reasoning in small LLMs. Zero pretraining leakage. Infinite synthetic data. Verifiable ground truth.

Spectral-R1 ablation candidate · NeurIPS 2026
02 / 07
The Problem

We can't study reasoning in a clean room

When you train an LLM on Python or natural language, you can't tell if it learned to reason or just memorized pretraining patterns.

01
Pretraining leakage
Models already know Python. Any capability could be retrieval, not reasoning.
02
No dense reward
CoT is unstructured text. No intermediate verifiable steps between prompt and answer.
03
OOD unclear
Hard to define out-of-distribution for natural language tasks.
04
Backtracking implicit
Search is hidden inside the model. Nothing to inspect or supervise directly.
03 / 07
The Language

Three primitives. Full Turing power.

GORE programs are explicit search trees. Every execution is a verifiable, inspectable graph of decisions.

# STEP — deterministic unification X = red # FORK — all branches live simultaneously ? { X = a | X = b | X = c } # CUT — prune branch on unification failure ! X = goal -> search
FORK (3 branches) branch[0] → STEP X = red ✓ branch[1] → STEP X = green ✓ branch[2] → CUT failed → backtrack
04 / 07
Real-World Use Cases

Encoding real tasks in GORE

MMReD · Dense Context Reasoning
Which room was empty the longest?
ICLR 2026
Natural task
Given 128 video frames of a building, find the room that had zero occupants for the longest consecutive span. Requires global attention — every frame is relevant.
GORE encoding
longest_empty(R): ? { R = room_a | R = room_b | R = room_c }; ! MaxSpan = max_empty(R) -> longest_empty
Multi-hop QA · NLP
Who founded the company that acquired X?
MuSiQue
Natural task
Decompose into: (1) find company that acquired X, (2) find founder of that company. Each step depends on the previous — classic backtracking chain.
GORE encoding
answer(Founder): Co = acquirer(x); ! Co = none -> answer; Founder = founded_by(Co)
Planning · Agent Reasoning
Find a path under constraints
Search
Natural task
Navigate a graph from A to D, avoiding blocked nodes. Must backtrack when hitting a dead end — the core structure of planning and constraint satisfaction.
GORE encoding
path(X, Y): ? { X = Y | Z = next(X); ! Z = blocked -> path; path(Z, Y) }
05 / 07
Connection to MMReD

Why GORE matters for dense context reasoning

MMReD showed that all current models collapse on dense context tasks — even at 0% on some subtasks at 128 observations. GORE lets us study why.

MMReD findingWhat GORE enables
Models fail to integrate global patternsFORK encodes all hypotheses simultaneously — we can train and measure this
NIAH performance doesn't predict dense reasoningGORE separates retrieval (STEP) from search (FORK+CUT) by construction
SFT/GRPO fail to generalize to longer contextOOD axis is explicit: train on depth 1–3, test on depth 4–5
Attention architecture is the bottleneckE3 spectral analysis measures this cleanly — no NL noise
Thesis

GORE is not just an ablation toy — it is a formal model of exactly the reasoning failure that MMReD diagnosed empirically. It gives us a controlled environment to fix what MMReD identified as broken.

Core Hypothesis

A micro-LLM trained on GORE traces will show better OOD generalization on unseen tree depths than a larger model trained on natural language CoT — because the inductive bias of the language matches the inductive bias of the task.
